# Design Patterns Every Developer Should Know

Design patterns are reusable solutions to recurring problems. They're the shared vocabulary of software architecture.

## Strategy Pattern

Encapsulate interchangeable behaviors. When you see long if-else chains, the Strategy pattern is usually the answer.

## Observer Pattern

When one object changes and others need to know, the Observer pattern creates a clean publish-subscribe relationship.

## Factory Pattern

Defer object creation to subclasses or factory methods. It decouples client code from concrete implementations.

Patterns are not rules — they're tools. Use them when they solve real problems.
